  • 速度要快；

3.3 解决方案——numpy的memmap

简介：

“内存映像文件是一种将磁盘上的非常大的二进制数据文件当做内存中的数组进行处理的方式。NumPy实现了一个类似于ndarray的memmap对象，它允许将大文件分成小段进行读写，而不是一次性将整个数组读入内存。memmap也拥有跟普通数组一样的方法，因此，基本上只要是能用于ndarray的算法就也能用于memmap。”

np.memmap作用_Mingsheng Zhang的博客-CSDN博客_numpy.memmap

调用：

fp = np.memmap(file_path, dtype=ubyte, mode = 'r+', offset = 0, shape = None, order = 'C')
'''
Args:
  file_path: 文件路径
  dtype: 数据类型
  mode:（与文件类似）
        r 只读
        r+ 读写
        w+ 创建或覆盖文件
        C 写时复制，会改变内存中的数据，但不会保存到磁盘。磁盘上的数据是只读的
  shape: 数组的形状，默认是一维的
'''

心得：

Memmap优势在于存大量数据，速度很快，可以调用np数组的索引、切片方法；

缺点在于存取的时候要记住数组形状，且形状比较固定，不能动态变化，且一不小心用错mode导致数据丢失，和文件一样

 

 

 

 

3. 给定句子的embedding，如何检索最近的k个近邻？

 

3.1要求:

  • 在检索阶段，给定一个句子的embedding，在上述23G文件（大约500w）里搜寻最近的K个近邻，默认欧式距离;
  • 考虑到规模太大，允许速度和精度之间有trade-off;
  • 返回结果包括近邻、距离，对返回结果按照距离做排序;

 

3.2 解决方案—— Meta Faiss

Faiss is a library for efficient similarity search and clustering of dense vectors. It contains algorithms that search in sets of vectors of any size, up to ones that possibly do not fit in RAM. It also contains supporting code for evaluation and parameter tuning.

Faiss is written in C++ with complete wrappers for Python. Some of the most useful algorithms are implemented on the GPU. It is developed by Facebook AI Research.

————https://faiss.ai/

3.4 Tutorial：

根据数据库建立Index ——> 给定查询向量search ——> 返回距离，对应的ID

本次介绍的三种Index(以欧式距离为例）:

 

IndexFlat：对查询向量做暴力检索

 

import numpy as np


d = 64                           # dimension
nb = 100000                      # database size
nq = 10000                       # nb of queries
np.random.seed(1234)             # make reproducible
xb = np.random.random((nb, d)).astype('float32')
xb[:, 0] += np.arange(nb) / 1000.
xq = np.random.random((nq, d)).astype('float32')
xq[:, 0] += np.arange(nq) / 1000.


import faiss                   # make faiss available
index = faiss.IndexFlatL2(d)   # build the index
print(index.is_trained)
index.add(xb)                  # add vectors to the index
print(index.ntotal)
print("==================================")
k = 4                          # we want to see 4 nearest neighbors
D, I = index.search(xb[:5], k) # sanity check
print("D:", D.shape, "I:", I.shape)
print(I)
print(D)
print("==================================")
D, I = index.search(xq, k)     # actual search
print("D:", D.shape, "I:", I.shape)
print(I[:5])                   # neighbors of the 5 first queries
print(I[-5:])                  # neighbors of the 5 last queries

 

 

IndexIVF（倒排文件）：

将原来的向量库建立N个中心，每个中心后面挂上相应的向量（此即倒排的含义），检索阶段，对于query向量，在指定的中心簇、附近的簇做暴力检索，如下图所示，在每个Cell内做检索:

import numpy as np
import time
d = 128                           # dimension
nb = 100000                      # database size
nq = 10000                       # nb of queries
np.random.seed(1234)             # make reproducible
xb = np.random.random((nb, d)).astype('float32')
xb[:, 0] += np.arange(nb) / 10000.
xq = np.random.random((nq, d)).astype('float32')
xq[:, 0] += np.arange(nq) / 10000.


import faiss


nlist = 100
k = 10
quantizer = faiss.IndexFlatL2(d)  # the other index
index = faiss.IndexIVFFlat(quantizer, d, nlist, faiss.METRIC_L2)
# here we specify METRIC_L2, by default it performs inner-product search


assert not index.is_trained
index.train(xb)                 # 获取聚类中心，需要先进行训练
assert index.is_trained


index.add(xb)                  # add may be a bit slower as well


print(f"nlist:{index.nlist}, nprobe:{index.nprobe}")


t0 = time.time()
D, I = index.search(xq, k)     # actual search
t1 = time.time()


print(I[-5:])    # neighbors of the 5 last queries
print(D[-5:])
print(f"search time = {t1-t0}")


print("\n======================================\n")


index.nprobe = 5              # default nprobe is 1, try a few more


print(f"nlist:{index.nlist}, nprobe:{index.nprobe}")


t0 = time.time()
D, I = index.search(xq, k)
t1 = time.time()


print(I[-5:])                  # neighbors of the 5 last queries
print(D[-5:])
print(f"search time = {t1-t0}")


print("\n======================================\n")


index.nprobe = 100              # default nprobe is 1, try a few more


print(f"nlist:{index.nlist}, nprobe:{index.nprobe}")


t0 = time.time()
D, I = index.search(xq, k)
t1 = time.time()


print(I[-5:])                  # neighbors of the 5 last queries
print(D[-5:])
print(f"search time = {t1-t0}")

IVFPQ [1]

采用量化（PQ）的方法，把每个向量转换成子向量，同时使用子向量的id代表每个向量，有效减小了存储空间。

  1. We split the original vector into several subvectors.
  2. For each set of subvectors, we perform a clustering operation — creating multiple centroids for each sub-vector set.
  3. In our vector of sub-vectors, we replace each sub-vector with the ID of it’s nearest set-specific centroid.

速度快，精度损失严重，一般用在工程里。

# single gpu
index_ivf = faiss.IndexIVFFlat(quantizer, d, nlist, faiss.METRIC_L2)
res = faiss.StandardGpuResources()  # use a single GPU
gpu_index_ivf = faiss.index_cpu_to_gpu(res, 0, index_ivf) # gpu id = 0


# multi gpu
co = faiss.GpuMultipleClonerOptions()
co.shard = True # 是否共享数据, Replication/Sharding
gpu_index_ivf = faiss.index_cpu_to_gpus_list(index_ivf, gpus=[0, 1, 2, 3], co=co)
# 注意是按照簇划分，不是均匀共享数据，所以一个簇数据过多可能导致单张GPU放不下
